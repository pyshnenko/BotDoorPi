# RASP - Система управления дверью через WebSocket

Приложение для управления GPIO пинами на Raspberry Pi через WebSocket соединение с удаленным сервером. Проект предназначен для управления открытием/закрытием двери через интеллектуальную систему.

## Функциональность

- **WebSocket клиент** - подключается к удаленному серверу для получения команд управления
- **GPIO управление** - контроль пинов 27 и 22 на Raspberry Pi (открытие/закрытие двери)
- **Сетевой мониторинг** - отслеживание состояния сетевого подключения
- **Автоматическое переподключение** - восстановление соединения при разрыве связи
- **Heartbeat (пульс)** - периодическая проверка живого соединения с сервером
- **Обновление приложения** - возможность загружения обновлений со статического хоста

## Требования

- **Node.js** v16+
- **Raspberry Pi** (для работы с GPIO) или Linux система
- **npm** или **yarn** для управления зависимостями

## Установка

### 1. Клонирование/загрузка проекта
```bash
cd путь/к/проекту
npm install
```

### 2. Конфигурация

Создайте файл `.env` в корне проекта:

```env
REMOTE_SERVER_URL=192.168.1.100
REMOTE_SERVER_PORT=8080
UPDATE_FILE_URL=https://example.com/update
```

**Переменные окружения:**
- `REMOTE_SERVER_URL` - IP адрес или хост WebSocket сервера
- `REMOTE_SERVER_PORT` - порт WebSocket сервера (по умолчанию: порт из переменной)
- `UPDATE_FILE_URL` - URL для загрузки обновлений приложения (по умолчанию: https://spamigor.ru)

## GPIO Конфигурация

Проект использует два GPIO пина на Raspberry Pi:

- **Pin 27** - Открытие двери (OPEN_PIN)
- **Pin 22** - Закрытие двери (CLOSE_PIN)

При запуске на Linux системе приложение инициализирует пины с помощью команды `pinctrl`:
```bash
pinctrl set 27 op dl  # Открытие
pinctrl set 22 op dl  # Закрытие
```

## WebSocket Протокол

Приложение подключается к WebSocket серверу и слушает входящие команды для управления GPIO пинами.

**Пример работы:**
1. Приложение подключается к серверу
2. Периодически отправляет heartbeat сигналы
3. Получает команды управления от сервера
4. Управляет GPIO пинами в соответствии с командами
5. При разрыве соединения - автоматическое переподключение

## Решение проблем

### Приложение не подключается к серверу
- Проверьте значения `REMOTE_SERVER_URL` и `REMOTE_SERVER_PORT` в файле `.env`
- Убедитесь, что сервер запущен и доступен
- Проверьте брандмауэр и правила доступа

### GPIO пины не инициализируются
- Приложение требует прав администратора на Raspberry Pi
- Убедитесь, что используется Linux система (GPIO доступен только на Linux)
- Проверьте, что `pinctrl` установлен и доступен в PATH

### Ошибка при импорте модулей
```bash
# Переустановите зависимости
rm -rf node_modules package-lock.json
npm install
```
